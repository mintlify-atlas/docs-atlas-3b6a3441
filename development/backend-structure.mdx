## Overview

The SESA backend is built with **FastAPI**, a modern Python web framework. It provides a RESTful API with automatic documentation, authentication, and database integration using SQLAlchemy ORM.

## Technology Stack

- **Python 3.14+** - Modern Python with latest features
- **FastAPI** - High-performance async web framework
- **SQLAlchemy 2.0+** - SQL toolkit and ORM
- **PyMySQL** - MySQL database connector
- **Pydantic** - Data validation using Python type hints
- **Bcrypt** - Password hashing
- **Uvicorn** - ASGI server for production
- **APScheduler** - Background task scheduling

## Application Entry Point

### main.py

The FastAPI application with CORS, routers, and lifecycle management:

```python app/main.py
import os
from contextlib import asynccontextmanager
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from dotenv import load_dotenv
from sqlalchemy import text
from apscheduler.schedulers.background import BackgroundScheduler

from app.db.database import engine
from app.routers.students import router as students_router
from app.routers import Listados
from app.routers import auth
from app.routers import mock_api
from app.routers import sync
from app.routers import catalogos
from app.routers.sync import tarea_automatica_sincronizacion

load_dotenv()

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup: Initialize background scheduler
    scheduler = BackgroundScheduler()
    scheduler.add_job(tarea_automatica_sincronizacion, 'interval', minutes=1)
    scheduler.start()
    print("Cronjob de Sincronización Iniciado (Corriendo cada 1 minuto)")

    # Test database connection
    try:
        with engine.connect() as connection:
            connection.execute(text("SELECT 1"))
        print("\n¡CONEXIÓN A BASE DE DATOS EXITOSA!\n")
    except Exception as e:
        print(f"\nERROR AL CONECTAR A LA BD: {e}\n")

    yield

    # Shutdown: Stop scheduler
    scheduler.shutdown()
    print("Cronjob de Sincronización Detenido")

app = FastAPI(
    title="SESA API",
    version="1.0.0",
    lifespan=lifespan
)

# CORS Configuration
origins_env = os.getenv("BACKEND_CORS_ORIGINS", "")
origins = [o.strip() for o in origins_env.split(",") if o.strip()]

if not origins:
    origins = [
        "http://localhost:5173",
        "http://127.0.0.1:5173",
        "http://localhost:5174",
        "http://127.0.0.1:5174",
    ]

print("CORS ORIGINS ACTIVOS:", origins)

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

# Include routers
app.include_router(students_router)
app.include_router(Listados.router)
app.include_router(auth.router)
app.include_router(mock_api.router)
app.include_router(sync.router)
app.include_router(catalogos.router)

@app.get("/")
def read_root():
    return {"message": "El sistema SESA está funcionando"}
```

**Key Features:**
- **Lifespan Management**: Handles startup/shutdown events
- **Background Scheduler**: Runs periodic synchronization tasks
- **CORS Middleware**: Enables cross-origin requests from frontend
- **Router Organization**: Modular endpoint groups
- **Database Health Check**: Verifies connection on startup

### Security

Password hashing and verification using bcrypt:

```python app/core/security.py
import bcrypt

def get_password_hash(password: str) -> str:
    """Hash a password using bcrypt"""
    pwd_bytes = password.encode('utf-8')
    salt = bcrypt.gensalt()
    hashed = bcrypt.hashpw(pwd_bytes, salt)
    return hashed.decode('utf-8')

def verify_password(plain_password: str, hashed_password: str) -> bool:
    """Verify a password against its hash"""
    try:
        pwd_bytes = plain_password.encode('utf-8')
        hash_bytes = hashed_password.encode('utf-8')
        return bcrypt.checkpw(pwd_bytes, hash_bytes)
    except Exception:
        return False
```

**Usage:**
- `get_password_hash()` - Hash passwords before storing
- `verify_password()` - Authenticate users during login

## Database Layer

### Database Connection

SQLAlchemy engine and session management:

```python app/db/database.py
from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, declarative_base
from app.core.config import DATABASE_URL

engine = create_engine(
    DATABASE_URL,
    pool_pre_ping=True  # Verify connections before using
)

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

Base = declarative_base()

def get_db():
    """Dependency for database sessions"""
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
```

**Key Features:**
- `pool_pre_ping`: Ensures connections are alive before use
- `get_db()`: Dependency injection for route handlers
- `Base`: Parent class for all ORM models

## Routers Layer

FastAPI routers organize endpoints by domain:

### Authentication Router

```python app/routers/auth.py
from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.orm import Session
from sqlalchemy import or_
from datetime import datetime

from app.db.database import get_db
from app.models.user import User
from app.schemas.auth import LoginRequest, UserResponse, PasswordChangeRequest
from app.core.security import verify_password, get_password_hash

router = APIRouter(prefix="/auth", tags=["auth"])

@router.post("/login", response_model=UserResponse)
def login(data: LoginRequest, db: Session = Depends(get_db)):
    """Authenticate user with identifier and password"""
    generic_error = "ID o contraseña incorrectos"

    # Support login by identifier or email
    user = (
        db.query(User)
        .filter(or_(User.identifier == data.identifier, User.email == data.identifier))
        .first()
    )

    if not user:
        raise HTTPException(status_code=401, detail=generic_error)

    if not verify_password(data.password, user.password_hash):
        raise HTTPException(status_code=401, detail=generic_error)

    # Update last login
    user.last_login = datetime.utcnow()
    db.commit()
    db.refresh(user)

    return user

@router.put("/change-password")
def change_password(data: PasswordChangeRequest, db: Session = Depends(get_db)):
    """Change user password with validation"""
    user = db.query(User).filter(User.identifier == data.identifier).first()
    if not user:
        raise HTTPException(status_code=404, detail="Usuario no encontrado")

    # Verify current password
    if not verify_password(data.current_password, user.password_hash):
        raise HTTPException(status_code=400, detail="La contraseña actual es incorrecta")

    # Confirm new password
    if data.new_password != data.confirm_password:
        raise HTTPException(status_code=400, detail="Las contraseñas no coinciden")

    # Prevent reusing same password
    if verify_password(data.new_password, user.password_hash):
        raise HTTPException(status_code=400, detail="La nueva contraseña no puede ser igual a la actual")

    # Update password
    user.password_hash = get_password_hash(data.new_password)
    user.is_temp_password = False

    db.commit()
    db.refresh(user)

    return {"message": "Contraseña actualizada exitosamente"}
```

**Router Features:**
- `prefix="/auth"` - All routes prefixed with `/auth`
- `tags=["auth"]` - Groups in OpenAPI docs
- `response_model` - Automatic validation and serialization
- `Depends(get_db)` - Dependency injection for database sessions

### Students Router

The students router handles student management operations. Here's a partial example:

```python app/routers/students.py (partial)
from fastapi import APIRouter, Depends, HTTPException, UploadFile, File
from sqlalchemy.orm import Session
from typing import List
import pandas as pd

from app.db.database import get_db
from app.models.student import Student
from app.schemas.student import StudentResponse, StudentCreate

router = APIRouter(prefix="/api/students", tags=["students"])

@router.get("/", response_model=List[StudentResponse])
def list_students(
    skip: int = 0,
    limit: int = 100,
    db: Session = Depends(get_db)
):
    """Get list of students with pagination"""
    students = db.query(Student).offset(skip).limit(limit).all()
    return students

@router.get("/{matricula}", response_model=StudentResponse)
def get_student(matricula: str, db: Session = Depends(get_db)):
    """Get student by matricula"""
    student = db.query(Student).filter(Student.matricula == matricula).first()
    if not student:
        raise HTTPException(status_code=404, detail="Estudiante no encontrado")
    return student

@router.post("/", response_model=StudentResponse)
def create_student(student: StudentCreate, db: Session = Depends(get_db)):
    """Create new student"""
    # Check if matricula already exists
    existing = db.query(Student).filter(Student.matricula == student.matricula).first()
    if existing:
        raise HTTPException(status_code=400, detail="Matrícula ya existe")
    
    # Create new student
    db_student = Student(**student.dict())
    db.add(db_student)
    db.commit()
    db.refresh(db_student)
    return db_student

@router.post("/import")
async def import_students(file: UploadFile = File(...), db: Session = Depends(get_db)):
    """Bulk import students from Excel file"""
    if not file.filename.endswith(('.xlsx', '.xls')):
        raise HTTPException(status_code=400, detail="Archivo debe ser Excel")
    
    # Read Excel file
    df = pd.read_excel(file.file)
    
    # Process each row
    imported = 0
    for _, row in df.iterrows():
        student = Student(
            matricula=row['matricula'],
            nombre=row['nombre'],
            apellido_paterno=row['apellido_paterno'],
            # ... more fields
        )
        db.add(student)
        imported += 1
    
    db.commit()
    return {"message": f"{imported} estudiantes importados"}
```

**Common Patterns:**
- Query parameters for filtering/pagination
- Path parameters for resource IDs
- Request body validation with Pydantic
- File uploads with `UploadFile`
- Bulk operations with pandas

## Best Practices

### 1. Use Dependency Injection

```python
# ✅ Good: Use Depends()
def get_students(db: Session = Depends(get_db)):
    pass

# ❌ Bad: Create session manually
def get_students():
    db = SessionLocal()
    # ...
    db.close()
```

### 2. Validate Input with Pydantic

```python
# ✅ Good: Use schemas
class StudentCreate(BaseModel):
    nombre: str
    email: EmailStr

@router.post("/students")
def create_student(student: StudentCreate):
    pass

# ❌ Bad: Use raw dict
@router.post("/students")
def create_student(student: dict):
    pass
```

### 3. Use Response Models

```python
# ✅ Good: Define response model
@router.get("/students", response_model=List[StudentResponse])
def get_students(db: Session = Depends(get_db)):
    return db.query(Student).all()

# ❌ Bad: Return raw SQLAlchemy objects
@router.get("/students")
def get_students(db: Session = Depends(get_db)):
    return db.query(Student).all()
```

### 4. Handle Errors Gracefully

```python
# ✅ Good: Specific error messages
@router.get("/students/{matricula}")
def get_student(matricula: str, db: Session = Depends(get_db)):
    student = db.query(Student).filter(Student.matricula == matricula).first()
    if not student:
        raise HTTPException(status_code=404, detail="Estudiante no encontrado")
    return student

# ❌ Bad: Generic errors
@router.get("/students/{matricula}")
def get_student(matricula: str, db: Session = Depends(get_db)):
    return db.query(Student).filter(Student.matricula == matricula).first()
```

## Development Commands

### Run Development Server

```bash
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**Options:**
- `--reload` - Auto-reload on code changes
- `--host 0.0.0.0` - Accept connections from any IP
- `--port 8000` - Listen on port 8000

### Run with Different Port

```bash
uv run uvicorn app.main:app --reload --port 8001
```

### Run in Production

```bash
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
```

**Production Options:**
- Remove `--reload` (slower in production)
- Add `--workers N` for multiple processes
- Use process manager like systemd or supervisor
