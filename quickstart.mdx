This guide will help you set up the Sistema de Evaluación y Seguimiento Académico (SESA) on your local development environment.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Python 3.14+** - Required for the FastAPI backend
- **Node.js 18+** - Required for the React frontend
- **Docker** - For running the MySQL database
- **Git** - For cloning the repository
- **uv** - Python package manager (recommended for dependency management)

<Note>
  SESA uses modern Python 3.14 features. Make sure your Python version is up to date.
</Note>

## Installation Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the SESA repository to your local machine:

    ```bash
    git clone <repository-url>
    cd sesa
    ```
  </Step>

  <Step title="Set Up the Database">
    Start the MySQL database using Docker:

    ```bash
    docker run -d \
      --name sesa-mysql \
      -e MYSQL_ROOT_PASSWORD=rootpassword \
      -e MYSQL_DATABASE=sesa_db \
      -e MYSQL_USER=sesa_user \
      -e MYSQL_PASSWORD=sesa_password \
      -p 3306:3306 \
      mysql:8.0
    ```

    <Note>
      The database will be accessible on `localhost:3306`. Update credentials as needed for your environment.
    </Note>
  </Step>

  <Step title="Configure Backend Environment">
    Navigate to the backend directory and create a `.env` file:

    ```bash
    cd backend
    ```

    Create a `.env` file with the following configuration:

    ```bash
    # Database Configuration
    MYSQL_USER=sesa_user
    MYSQL_PASSWORD=sesa_password
    MYSQL_SERVER=localhost
    MYSQL_PORT=3306
    MYSQL_DB=sesa_db

    # CORS Configuration (optional)
    BACKEND_CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
    ```

    <Warning>
      Never commit the `.env` file to version control. Keep your database credentials secure.
    </Warning>
  </Step>

  <Step title="Install Backend Dependencies">
    Install Python dependencies using uv (recommended) or pip:

    <CodeGroup>
    ```bash uv (Recommended)
    # Create virtual environment
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate

    # Install uv if not already installed
    pip install uv

    # Install dependencies
    uv pip install -e .
    ```

    ```bash pip
    # Create virtual environment
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate

    # Install dependencies
    pip install fastapi[standard-no-fastapi-cloud-cli] uvicorn sqlalchemy pymysql \
                python-dotenv bcrypt passlib[bcrypt] pandas openpyxl \
                requests python-multipart cryptography
    ```
    </CodeGroup>

    The backend uses the following key dependencies:
    - **FastAPI** - Modern web framework for building APIs
    - **SQLAlchemy** - ORM for database operations
    - **PyMySQL** - MySQL database driver
    - **Bcrypt** - Password hashing and verification
    - **Pandas & OpenPyXL** - Data processing and Excel export
    - **APScheduler** - Background task scheduling
  </Step>

  <Step title="Run Database Migrations">
    Initialize the database schema:

    ```bash
    # Make sure you're in the backend directory with venv activated
    python -m app.db.init_db  # Or run your migration script
    ```

    <Note>
      If migrations aren't set up yet, SQLAlchemy will create tables automatically on first run based on your models.
    </Note>
  </Step>

  <Step title="Start the Backend Server">
    Run the FastAPI development server:

    ```bash
    uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
    ```

    You should see output indicating:
    - Database connection successful
    - Cronjob synchronization started (runs every 1 minute)
    - Server running at `http://localhost:8000`

    Visit `http://localhost:8000/docs` to explore the interactive API documentation.
  </Step>

  <Step title="Configure Frontend Environment">
    Open a new terminal and navigate to the frontend directory:

    ```bash
    cd frontend
    ```

    Create a `.env` file with the backend API URL:

    ```bash
    VITE_API_URL=http://localhost:8000
    ```
  </Step>

  <Step title="Install Frontend Dependencies">
    Install Node.js dependencies using npm:

    ```bash
    npm install
    ```

    The frontend uses:
    - **React 19** - UI library
    - **Vite** - Fast build tool and dev server
    - **Tailwind CSS 4** - Utility-first CSS framework
    - **Axios** - HTTP client for API requests
    - **React Router** - Client-side routing
    - **SweetAlert2** - Beautiful alert dialogs
  </Step>

  <Step title="Start the Frontend Server">
    Run the Vite development server:

    ```bash
    npm run dev
    ```

    The frontend will be available at `http://localhost:5173`
  </Step>

  <Step title="First Login">
    Open your browser and navigate to `http://localhost:5173`

    Default login credentials (if seeded):
    - **ID/Email**: Check your database or seed file
    - **Password**: Default temporary password

    <Note>
      On first login with a temporary password, you'll be prompted to change it immediately.
    </Note>
  </Step>
</Steps>

## Verify Installation

To verify everything is working correctly:

1. **Backend Health Check**: Visit `http://localhost:8000/` - you should see:
   ```json
   {"message": "El sistema SESA está funcionando"}
   ```

2. **API Documentation**: Visit `http://localhost:8000/docs` for interactive API docs

3. **Frontend**: Visit `http://localhost:5173` - you should see the SESA login page

4. **Database Connection**: Check the backend terminal for "¡CONEXIÓN A BASE DE DATOS EXITOSA!"

5. **Background Jobs**: Verify "Cronjob de Sincronización Iniciado" appears in backend logs

## Common Issues

<AccordionGroup>
  <Accordion title="Database Connection Failed">
    - Ensure Docker container is running: `docker ps | grep sesa-mysql`
    - Verify `.env` credentials match the Docker container configuration
    - Check if port 3306 is already in use: `lsof -i :3306`
  </Accordion>

  <Accordion title="Port Already in Use">
    If port 8000 or 5173 is already in use:
    
    **Backend**: Run on a different port
    ```bash
    uvicorn app.main:app --reload --port 8001
    ```
    
    **Frontend**: Vite will automatically try the next available port (5174, 5175, etc.)
  </Accordion>

  <Accordion title="CORS Errors">
    If you see CORS errors in the browser console:
    - Ensure frontend URL is in `BACKEND_CORS_ORIGINS` in backend `.env`
    - Restart the backend server after changing CORS configuration
  </Accordion>

  <Accordion title="Module Import Errors">
    - Ensure virtual environment is activated: `source venv/bin/activate`
    - Reinstall dependencies: `uv pip install -e .`
    - Check Python version: `python --version` (must be 3.14+)
  </Accordion>
</AccordionGroup>

## Development Tools

### Hot Reload

Both servers support hot reload:
- **Backend**: FastAPI auto-reloads on Python file changes
- **Frontend**: Vite provides instant HMR (Hot Module Replacement)

### Database Management

Access the MySQL database directly:

```bash
docker exec -it sesa-mysql mysql -u sesa_user -p
# Enter password: sesa_password
```

### API Testing

Use the interactive API docs at `http://localhost:8000/docs` or tools like:
- **curl** - Command-line HTTP client
- **Postman** - API development platform
- **HTTPie** - User-friendly CLI HTTP client

## Production Deployment

<Warning>
  This quickstart is for development only. For production deployment:
  - Use secure passwords and environment variables
  - Enable HTTPS/TLS encryption
  - Configure proper CORS policies
  - Use a production WSGI server (not uvicorn --reload)
  - Set up proper database backups
  - Configure logging and monitoring
</Warning>

For production deployment, SESA uses:
- **Netlify** - Frontend hosting with CI/CD
- **MySQL** - Production database
- **Tailscale** - Secure Mesh VPN for database access
