## Overview

The student management system provides comprehensive functionality for registering, listing, and updating student profiles. Each student is assigned a unique auto-generated matricula (student ID) and can have associated files like profile photos and certificates.

## Student Registration

### Manual Registration

Students can be registered individually through the `/alumnos/register` endpoint with multipart form data including personal information and file uploads.

```python
@router.post("/register", status_code=status.HTTP_201_CREATED)
def register_student(
    student_data: str = Form(...),
    foto_perfil: UploadFile = File(...),
    certificado: UploadFile = File(None),
    db: Session = Depends(get_db)
):
    try:
        data_dict = json.loads(student_data)
        student_in = StudentCreate(**data_dict)
    except Exception as e:
        raise HTTPException(status_code=400, detail=f"Error en datos: {str(e)}")
```

### Automatic Matricula Generation

The system automatically generates sequential matriculas based on the highest existing value in the database:

```python
max_m = db.query(func.max(Student.matricula)).scalar()
final_matricula = str(int(max_m) + 1) if max_m and max_m.isdigit() else "20240001"
```

<Note>
The matricula is an 8-digit numeric identifier that serves as the primary key. If no students exist, it defaults to "20240001".
</Note>

### Duplicate Detection

Before creating a student, the system validates that CURP and email are unique:

```python
alumno_existente = db.query(Student).filter(
    (Student.curp == student_in.curp) |
    (Student.email_personal == student_in.email_personal)
).first()

if alumno_existente:
    if alumno_existente.curp == student_in.curp:
        raise HTTPException(status_code=400, detail="Esta CURP ya se encuentra registrada en el sistema.")
    if alumno_existente.email_personal == student_in.email_personal:
        raise HTTPException(status_code=400, detail="Este correo personal ya está en uso por otro alumno.")
```

### Search Implementation

The search term is applied to both matricula and full name:

```python
if busqueda:
    termino = f"%{busqueda}%"
    query = query.filter(

## Student Model Structure

class Student(Base):
    __tablename__ = "students"

    matricula = Column(String(20), primary_key=True, index=True)
    nombre = Column(String(100), nullable=False)
    apellido_paterno = Column(String(100), nullable=False)
    apellido_materno = Column(String(100), nullable=False)
    curp = Column(String(18), nullable=False, unique=True)
    foto_id = Column(BigInteger, ForeignKey("files.id"), nullable=True)
    email_personal = Column(String(150), nullable=False)
    email_institucional = Column(String(150), nullable=True)
    origin_school_id = Column(Integer, ForeignKey("origin_schools.id"), nullable=True)
    promedio_procedencia = Column(Integer, nullable=False)
    certificado_id = Column(BigInteger, ForeignKey("files.id"), nullable=True)
    career_id = Column(Integer, ForeignKey("careers.id"), nullable=False)
    cuatrimestre_actual = Column(Integer, nullable=False, server_default=text("1"))
    status_id = Column(Integer, ForeignKey("student_statuses.id"), nullable=False)
```

## Key Features

- **Automatic ID Generation**: Sequential 8-digit matriculas
- **Duplicate Prevention**: CURP and email uniqueness validation
- **File Management**: Database storage for photos and certificates
- **Advanced Search**: Filter by name, matricula, career, and semester
- **Pagination**: Configurable page size with total count
- **Profile Updates**: Modify student data while preserving academic history
- **Address Management**: Separate address storage linked to student
