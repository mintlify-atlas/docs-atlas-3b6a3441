---
title: 'Change Password'
api: 'PUT /auth/change-password'
description: 'Change a user password with validation and confirmation'
---

## Overview

The change password endpoint allows users to update their password securely. It validates the current password, ensures the new password matches the confirmation, and prevents users from reusing their current password.

Upon successful password change, the `is_temp_password` flag is set to `false`, indicating that the user is no longer using a temporary password.

## Request Body

<ParamField body="identifier" type="string" required>
  User's unique identifier (ID). Used to locate the user account.
</ParamField>

<ParamField body="current_password" type="string" required>
  The user's current password. Must match the existing password for authentication.
</ParamField>

<ParamField body="new_password" type="string" required>
  The new password to set. Must be at least 8 characters long.
  
  **Minimum length:** 8 characters
</ParamField>

<ParamField body="confirm_password" type="string" required>
  Confirmation of the new password. Must match `new_password` exactly.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message confirming the password was updated.
</ResponseField>

## Validation Rules

The endpoint enforces the following validation rules:

1. **User Existence**: The user identifier must exist in the system
2. **Current Password Verification**: The provided current password must be correct
3. **Password Confirmation**: The new password must match the confirmation password
4. **Password Change Required**: The new password cannot be the same as the current password
5. **Minimum Length**: The new password must be at least 8 characters long

## Behavior

- Verifies the current password against the stored password hash
- Validates that the new password matches the confirmation
- Ensures the new password is different from the current password
- Hashes the new password before storing it
- Sets `is_temp_password` to `false` after successful password change
- Commits changes to the database immediately

## Request Example

```bash
curl -X PUT http://localhost:8000/auth/change-password \
  -H "Content-Type: application/json" \
  -d '{
    "identifier": "USR001",
    "current_password": "oldPassword123",
    "new_password": "newSecurePass456",
    "confirm_password": "newSecurePass456"
  }'
```

## Response Examples

### Success Response (200 OK)

```json
{
  "message": "Contraseña actualizada exitosamente"
}
```

### Error Responses

#### User Not Found (404 Not Found)

```json
{
  "detail": "Usuario no encontrado"
}
```

#### Incorrect Current Password (400 Bad Request)

```json
{
  "detail": "La contraseña actual es incorrecta"
}
```

#### Passwords Don't Match (400 Bad Request)

```json
{
  "detail": "Las contraseñas no coinciden"
}
```

#### New Password Same as Current (400 Bad Request)

```json
{
  "detail": "La nueva contraseña no puede ser igual a la actual"
}
```

#### Validation Error - Password Too Short (422 Unprocessable Entity)

```json
{
  "detail": [
    {
      "loc": ["body", "new_password"],
      "msg": "ensure this value has at least 8 characters",
      "type": "value_error.any_str.min_length",
      "ctx": {
        "limit_value": 8
      }
    }
  ]
}
```

## Error Codes

| Status Code | Description |
|-------------|-------------|
| 200 | Password changed successfully |
| 400 | Validation error (incorrect current password, passwords don't match, or same password) |
| 404 | User not found |
| 422 | Request validation error (missing fields or invalid format) |
