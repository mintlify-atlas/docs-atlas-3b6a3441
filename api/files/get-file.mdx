## Description

This endpoint retrieves file content stored in the database, such as student profile photos, certificates, and other uploaded documents. The file is returned as binary content with the appropriate MIME type.

## Path Parameters

<ParamField path="file_id" type="integer" required>
  The unique identifier of the file to retrieve
</ParamField>

## Response

Returns the raw binary content of the file with appropriate headers.

### Response Headers

<ResponseField name="Content-Type" type="string" required>
  The MIME type of the file (e.g., `image/jpeg`, `image/png`, `application/pdf`)
</ResponseField>

<ResponseField name="Content-Disposition" type="string" required>
  Indicates how the file should be displayed. Format: `inline; filename="original_filename.ext"`
  
  The `inline` directive tells the browser to display the file in the browser window when possible (e.g., images, PDFs)
</ResponseField>

### Response Body

Binary file content (bytes)

## Error Responses

<ResponseField name="404 Not Found" type="object">
  Returned when the file ID does not exist in the database
  
  ```json
  {
    "detail": "Archivo no encontrado"
  }
  ```
</ResponseField>

## Example Request

```bash
curl -X GET "http://localhost:8000/alumnos/archivos/123" \
  -H "accept: application/json" \
  -o downloaded_file.jpg
```

## Example Response Headers

```http
HTTP/1.1 200 OK
Content-Type: image/jpeg
Content-Disposition: inline; filename="student_photo_12345.jpg"
Content-Length: 245832
```

## Common File Types

| File Type | MIME Type | Typical Use |
|-----------|-----------|-------------|
| JPEG Image | `image/jpeg` | Student profile photos |
| PNG Image | `image/png` | Student profile photos |
| PDF Document | `application/pdf` | Academic certificates, transcripts |

## Use Cases

### Display Student Profile Photo
```html
<img src="http://localhost:8000/alumnos/archivos/123" 
     alt="Student Photo" />
```

### Embed Certificate PDF
```html
<iframe src="http://localhost:8000/alumnos/archivos/456" 
        width="100%" height="600px"></iframe>
```

### Download File with JavaScript
```javascript
fetch('http://localhost:8000/alumnos/archivos/123')
  .then(response => response.blob())
  .then(blob => {
    const url = window.URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = 'student_document.pdf';
    a.click();
  });
```

## Security Considerations

<Warning>
  This endpoint currently does not implement authentication or authorization. In production:
  - Add authentication to verify the user is logged in
  - Implement authorization to ensure users can only access files they have permission to view
  - Consider rate limiting to prevent abuse
  - Validate file_id to prevent SQL injection (already handled by FastAPI's type validation)
</Warning>

## Implementation Details

- Files are stored in the `files` table with columns: `id`, `file_name`, `mime_type`, `size_bytes`, `file_content`
- File content is stored as `BYTEA` (binary data) in the database
- The response uses FastAPI's `Response` class to return binary content directly
- The `Content-Disposition: inline` header allows browsers to display compatible files (images, PDFs) directly
